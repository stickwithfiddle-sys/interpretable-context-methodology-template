# Style Guide

Use this file for stable writing, formatting, and presentation preferences that should persist across runs.

## Default Tone

- Clear, concrete, and low-drama.
- Prefer direct verbs over abstract nouns.
- Separate facts, assumptions, and recommendations.

## Output Formatting

- Use markdown headings to match the stage contract.
- Use tables for stage maps, input inventories, and review gates.
- Keep action items short and assignable.

## Project-Specific Preferences

Add examples, voice rules, naming conventions, and formatting requirements here after the first few runs reveal what should be stable.
