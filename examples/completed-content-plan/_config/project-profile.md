# Project Profile

Project: Content Plan Example

Slug: content-plan-example

Created: 2026-05-30

## Owner

- Name: Hobo
- Role: Maintainer and workflow reviewer
- Decision authority: Approves stage maps and final scaffold decisions

## Project Aim

Design a repeatable ICM workspace that turns rough content inputs into a weekly content plan with clear review gates and traceable decisions.

## Users And Stakeholders

- Primary users: solo builders, small product teams, and technical writers
- Secondary users: reviewers who approve topics before drafting
- Reviewers or approvers: project owner
- People affected by errors: readers, customers, and contributors whose work is represented

## Constraints

- Budget: use local files and ordinary coding-agent tools first
- Timeline: the workflow should fit a weekly planning cadence
- Technical environment: markdown, git, and local validation scripts
- Legal, privacy, compliance, or safety constraints: do not publish private notes or unsourced claims
- Tools or vendors to avoid: avoid provider-specific requirements in the core workflow

## Definition Of Done

The workspace is ready when the stage map, scaffold plan, setup questionnaire, and validation report are reviewed and accepted.
