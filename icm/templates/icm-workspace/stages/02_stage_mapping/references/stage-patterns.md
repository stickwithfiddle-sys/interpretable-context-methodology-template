# Stage Patterns

Use these patterns when designing project-specific stages.

| Pattern | Use When | Typical Output |
| --- | --- | --- |
| Intake | A rough request needs structure | `project-brief.md` |
| Research | Source material needs synthesis | `research-summary.md` |
| Planning | Work needs decomposition | `plan.md` or `stage-map.md` |
| Drafting | A human-readable artifact must be created | `draft.md` |
| Review | An artifact needs critique or QA | `review-report.md` |
| Assembly | Approved parts become a deliverable | `final-spec.md` |
| Validation | Outputs need consistency checks | `validation-report.md` |

Prefer fewer stages until there is a real review gate or context boundary that justifies another folder.
